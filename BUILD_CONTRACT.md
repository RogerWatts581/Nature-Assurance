# Build Contract — MVP Version 1.0

Frozen for implementation on 15 July 2026.

The constitutional baseline was formally frozen on 19 July 2026 in [`CONSTITUTION.md`](./CONSTITUTION.md). This Build Contract implements that baseline; it does not amend it.

## Implementation rules

1. No new constitutional concepts during implementation. Classify any problem first as an implementation defect, then a specification defect, and only finally a constitutional defect.
2. Every feature must trace to an acceptance test. Features that satisfy none belong in the deferred list.

## Acceptance tests

| ID | Test |
|---|---|
| AT-01 | Every substantive proposition shows its evidence status. |
| AT-02 | The unsupported causal transition is visibly identified. |
| AT-03 | The original wording cannot be authorised for public use. |
| AT-04 | Challenge Review cannot approve or silently alter anything. |
| AT-05 | Human refusal and purpose-limited approval are recorded. |
| AT-06 | The Voice cannot repeat or strengthen the refused allegation. |
| AT-07 | New evidence creates a linked version while preserving the old one. |
